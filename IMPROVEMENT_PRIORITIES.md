# 개선 우선순위 가이드

## 🔴 높은 우선순위 (즉시 개선)

### 1. alert() → Toast로 변경
**위치**: `skinfront/app/auth/signup/components/SignupForm.tsx:226`
**문제**: `alert()` 사용으로 인한 나쁜 UX
**해결**: Toast 컴포넌트로 변경
```typescript
// 현재
alert('회원가입이 완료되었습니다! 이메일을 확인해주세요...')

// 개선
toast.success('회원가입이 완료되었습니다! 이메일을 확인해주세요.')
```

### 2. 접근성 (A11y) 개선
**현재 상태**: aria-label, role 등 접근성 속성 부족
**필요한 작업**:
- 모든 버튼에 `aria-label` 추가
- 모달에 `role="dialog"`, `aria-modal="true"` 추가
- 키보드 네비게이션 지원 (Tab, Enter, Esc)
- 색상 대비 비율 확인 (4.5:1 이상)

### 3. 에러 처리 개선
**현재 상태**: 네트워크 에러 시 수동 재시도만 가능
**필요한 작업**:
- 네트워크 에러 자동 재시도 (exponential backoff)
- 재시도 횟수 표시
- "다시 시도" 버튼 추가

### 4. 이미지 전처리 (성능/비용 절감)
**현재 상태**: 원본 이미지 그대로 업로드
**필요한 작업**:
- 클라이언트 사이드 리사이즈 (최대 1024px)
- WebP 형식 변환 (지원하는 경우)
- EXIF 데이터 제거 (개인정보 보호)
- WebWorker로 메인 스레드 블로킹 방지

## 🟡 중간 우선순위 (단기 개선)

### 5. 업로드 진행도 표시
**현재 상태**: 업로드 진행 상황 불명확
**필요한 작업**:
- 업로드 진행률 바 표시
- 예상 시간 표시
- 취소 버튼 추가

### 6. 이미지 품질 피드백
**현재 상태**: 업로드 후에만 품질 확인
**필요한 작업**:
- 클라이언트 사이드 품질 검사 (조명, blur 감지)
- 실시간 피드백 ("조명이 어둡습니다")
- 재촬영 권장 메시지

### 7. 분석 결과 UI 개선
**현재 상태**: 기본적인 결과 표시
**필요한 작업**:
- 레이더 차트 또는 수평 바 차트
- 히트맵 오버레이 슬라이더
- 색상 코딩된 배지 (Good / Needs attention / High priority)
- "왜 이 시술이 추천되는지" 설명 추가

### 8. 테스트 커버리지 확대
**현재 상태**: 일부 컴포넌트만 테스트
**필요한 작업**:
- 모든 훅에 대한 단위 테스트
- 주요 컴포넌트 통합 테스트
- E2E 테스트 시나리오 확대
- CI/CD에 테스트 자동화 추가
